# User Stories - Student Hub App

These user stories are based upon the 6 responses from the student survey.

## MVP Sprint 1: Mvp sprint 1 shows core features need right now

### Story 1: Grade Access
As a student, What I really want is to view my module grades in one place or a website so that I can track my progress.

Evidence: 50% most needed feature; open feedback: "easily view grades", "shows all grades for a module" |
Priority: P0 

### Story 2: Assignment Deadline
> As a student, I want to see all my assignment due dates so that I never miss a deadline.

| **Evidence** | 60% struggle with due dates (highest pain point) |
| **Priority** | P0 |

### Story 3: Useful Notifications
> As a student, I want useful notifications about important updates so that I don't miss critical information.

| **Evidence** | 33.3% most needed feature; open feedback: "give useful notifications" |
| **Priority** | P0 |

## MVP Sprint 2: Supporting Features (Should Have)

*These address significant pain points and improve the experience.*

### Story 4: Staff Contact
> As a student, I want to easily contact my lecturers so that I can get help when I need it.

| **Evidence** | 40% struggle contacting staff |
| **Priority** | P1 |

### Story 5: University Updates
> As a student, I want all university updates in one place so that I don't miss important announcements.

| **Evidence** | Open feedback: "easy access to university updates" |
| **Priority** | P1 |

## MVP Sprint 3: Enhancement Features (Nice to Have)

*These add value but are not essential for launch.*

### Story 6: Campus Navigation
> As a student, I want to find my way around campus so that I can locate buildings and rooms easily.

| **Evidence** | Open feedback: "show where everything is in the campus" |
| **Priority** | P2 |

### Story 7: Timetable Updates
> As a student, I want to receive timetable change notifications so that I don't show up to the wrong room.

| **Evidence** | 16.7% don't find out about changes |
| **Priority** | P2 |

---

## Future Releases (Post-MVP)

*These will be considered after launch based on feedback.*

### Story 8: Events Discovery
> As a student, I want to find campus events so that I can get involved in university life.

| **Evidence** | 20% struggle finding events |
| **Priority** | P3 |

### Story 9: Support Services
> As a student, I want to access wellbeing and finance support so that I can get help when needed.

| **Evidence** | 20% struggle accessing support |
| **Priority** | P3 |

### Story 10: Rewards
> As a student, I want to earn rewards for using the app so that I feel motivated.

| **Evidence** | 16.7% requested rewards |
| **Priority** | P4 |

## MVP Delivery Roadmap

| Sprint | Focus | Features | Rationale |
|--------|-------|----------|----------|
| **Sprint 1** | Core Needs | Grade Access, Assignment Deadlines, Notifications | Addresses highest demand (50%) and biggest pain point (60%) |
| **Sprint 2** | Communication | Staff Contact, University Updates | Tackles 40% pain point and feedback themes |
| **Sprint 3** | Enhancements | Campus Navigation, Timetable Updates | Adds value for remaining user needs |
| **Future** | Expansion | Events, Support Services, Rewards | Post-launch based on further feedback |
